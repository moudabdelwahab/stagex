# دليل نظام تتبع الأخطاء المركزي (Centralized Error Logging System)

تم بناء هذا النظام ليكون بديلاً داخلياً لخدمات مثل Sentry، حيث يعتمد بالكامل على **Supabase** لتخزين وإدارة الأخطاء البرمجية التي تحدث في الموقع.

## المكونات الرئيسية

### 1. قاعدة البيانات (Supabase)
تم إنشاء جدول باسم `site_errors` لتخزين كافة تفاصيل الأخطاء.
- **الملف:** `error_logging_setup.sql`
- **المميزات:** تفعيل سياسات الحماية (RLS) لضمان أن المسؤولين فقط هم من يمكنهم استعراض الأخطاء، بينما يمكن للجميع (Tracker) إرسالها.

### 2. متتبع الأخطاء (Frontend Tracker)
ملف JavaScript خفيف يتم تحميله في كافة صفحات الموقع لالتقاط الأخطاء تلقائياً.
- **الملف:** `error-tracker.js`
- **المهام:**
    - التقاط أخطاء التشغيل (Runtime Errors).
    - التقاط الوعود غير المعالجة (Unhandled Promise Rejections).
    - التقاط أخطاء الشبكة (Fetch Errors).
    - تصفية أخطاء المتصفح والإضافات (Extensions) لتقليل الضجيج.
    - منع الإغراق (Rate Limiting & Debounce).

### 3. لوحة التحكم الإدارية (Admin Dashboard)
واجهة مخصصة للمسؤولين لمتابعة الأخطاء وحلها.
- **الملف:** `admin/errors.html`
- **المميزات:**
    - عرض الأخطاء بترتيب زمني.
    - فلترة حسب النوع (JS, Promise, Network) أو الحالة (New, Resolved, Archived).
    - بحث متقدم في الرسائل والـ Stack Trace.
    - تحديث لحظي (Real-time) عند وقوع أي خطأ جديد.

## كيفية الدمج في المشروع

النظام مدمج بالفعل في الملفات الرئيسية، ولكن إليك ملخص للخطوات التي تم اتخاذها:

1. **إضافة السكريبت:** تم إضافة `<script src="/error-tracker.js"></script>` في رأس (Head) الملفات الرئيسية (`index.html`, `admin-dashboard.html`, `customer-dashboard.html`).
2. **تحديث القائمة الجانبية:** تم إضافة زر "مشاكل الموقع" في `assets/components/sidebar.html` يظهر فقط للمسؤولين.
3. **خدمة البيانات:** تم إنشاء `error-service.js` للتعامل مع عمليات جلب وتحديث البيانات من Supabase.

## ملاحظات أمنية
- النظام لا يقوم بتسجيل أي بيانات حساسة (مثل كلمات المرور).
- يتم استخدام مفتاح Supabase Anon Key للإرسال، بينما تتطلب لوحة التحكم صلاحيات Admin (JWT Role).
