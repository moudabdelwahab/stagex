# دليل إدارة API و Memory Firewall - منصة مدعوم

لقد تم إضافة نظام متقدم لإدارة مفاتيح الـ API وجدار حماية الذاكرة في لوحة تحكم `chat-admin`.

## 1. إدارة مفاتيح API (API Management)
يمكنك الآن توليد مفاتيح API لربط البوت بمواقع خارجية مع ميزات متقدمة:
- **توليد مفاتيح متعددة**: لكل موقع أو تطبيق مفتاح خاص به.
- **حالات التعطيل الذكي (Soft Disable)**:
    - `active`: المفتاح يعمل بكامل صلاحياته.
    - `read_only`: يسمح فقط بقراءة البيانات ولا يسمح بإرسال رسائل جديدة.
    - `rate_limited`: يطبق قيوداً مشددة على عدد الطلبات (لحماية السيرفر).
    - `maintenance`: يعطل المفتاح مؤقتاً مع إظهار حالة صيانة.
- **الصلاحيات (Permissions)**:
    - `chat:send`: القدرة على إرسال رسائل.
    - `memory:read`: القدرة على قراءة سياق المحادثة من الذاكرة.
    - `memory:write`: القدرة على تحديث الذاكرة (يتطلب حذر).

## 2. جدار حماية الذاكرة (Memory Firewall)
نظام لحماية ذاكرة البوت من "التسميم المعلوماتي" (Memory Poisoning) أو التعلم الخاطئ:
- **قواعد الكتابة (Rules)**: تحديد من يمكنه الكتابة في الذاكرة ومتى.
- **منع الكلمات الحساسة (Deny Keywords)**: منع حفظ أي معلومات حساسة مثل كلمات المرور أو التوكنات في الذاكرة.
- **الحد الأقصى للطول (Max Length)**: منع الرسائل الطويلة جداً التي قد تستنزف الذاكرة.

## 3. التعليمات البرمجية (للمطورين)
عند استدعاء الـ API من موقع خارجي، يجب تضمين المفتاح في الـ Header:
```http
Authorization: Bearer YOUR_API_KEY
```

سيقوم النظام تلقائياً بالتحقق من حالة المفتاح وصلاحياته قبل تنفيذ أي عملية.
